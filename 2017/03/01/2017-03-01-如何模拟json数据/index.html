三、各自优缺点

Charles优点

  • 在项目中不用考虑接口地址
  • 实时观察网络请求情况

Charles缺点

  • 依靠mock 模拟数据，多接口需要模拟多个
  • 需要每次拦截对应的请求URL，再跳转

json-server优点

  • 操作简单，修改简单

json-server缺点

  • 暂时没发现
